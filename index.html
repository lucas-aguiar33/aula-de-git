Git & GitHub

Tudo o que você precisa saber para versionar seus códigos de maneira simples e profissional

Comandos do Git

Git Init

Inicia um novo repositório git local na pasta atual.

git init

Git Status

Verifica o status atual do repositório, incluindo arquivos modificados e arquivos na área de stage.

git status

Git Add

Adiciona arquvios selecionados para área de stage.

git add .

Git Commit

Grava alterações feitas na área de stage como uma nova versão do código.

git commit -m "mensagem sobre o que está sendo enviado"

Git Log

Ver histórico de commits.

git log --all

Git Checkout

Usado para alternar entre branches, criar novos branches ou mudar para uma versão anterior do código em uma branch existente.

Alternar entre as branches:

git checkout nome da branch

Criar nova branch:

git checkout -b nome da branch

Voltar para alguma versão:

git checkout id da versão

* Quando criamos uma branch, muda direto para ela. Mas, para ter certeza, rodar o comando git log.

* Criar branch a partir do ponto em que queremos trabalhar em aglo do projeto sem afetar o funcionamento da main. Pegar sempre a versão atualizada.

Git Branch

Para ver quantas branches tem dentro do git e em qual está

git branch

Para remover uma branch:

git branch -d nome da branch
